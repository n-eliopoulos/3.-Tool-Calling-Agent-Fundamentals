Observability and Debugging

Agents are hard to debug because behavior is non-deterministic. The same input can produce different tool call sequences. Structured logging is your primary debugging tool.

What to log at each iteration

log_entry = {
    "iteration": i,
    "thought": extract_thought(response),  # Claude's reasoning text
    "action": block.name,                   # Tool called
    "tool_args": block.input,               # Arguments passed
    "result_summary": result[:200],         # Truncated result
    "state_snapshot": dict(state),          # Current state
    "tokens": {
        "input": response.usage.input_tokens,
        "output": response.usage.output_tokens,
        "cumulative": total_tokens,
    },
    "trace_id": trace_id,                   # Links all iterations
}

Debugging workflow

  1. Reproduce: Use the trace_id to find all log entries for a run.
  2. Read the thoughts: Claude's reasoning text explains why it chose each action.
  3. Check the results: Did the tool return what Claude expected?
  4. Find the divergence: Where did Claude's reasoning go wrong?
Try it: Add structured logging to the agent. Run it, then inspect the log to understand its reasoning at each step. Identify where it made good vs. questionable decisions.
If you can't see what the agent is thinking at each step, you can't debug it. Structured logging isn't optional for production agents — it's the foundation of observability.
