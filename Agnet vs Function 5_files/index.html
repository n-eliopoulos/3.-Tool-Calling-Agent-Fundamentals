Module 2: Agent vs. Function

When Functions Break Down

Smart functions fail when the task requires capabilities they don't have: multi-step reasoning with branching, adapting strategy based on intermediate results, exploring multiple approaches, or maintaining state across many interactions.

Signs you need more than a function

  • Branching logic based on results: "If the firm has violations, check each one. If any are recent, pull the full case file." The next step depends on what the previous step found.
  • Unknown number of steps: "Investigate this firm thoroughly." You don't know in advance how many tools you'll need.
  • Strategy adaptation: "If the compliance check fails, try looking at the parent company instead." The approach changes based on what happens.
  • Cross-referencing: "Find all firms connected to this individual and check each one." The scope expands based on intermediate results.

The gray zone

Many tasks sit in a gray zone. A function with 2 rounds of tool calling might work. Or you might need a 3-iteration bounded agent. The decision framework in the next slide helps you choose.
