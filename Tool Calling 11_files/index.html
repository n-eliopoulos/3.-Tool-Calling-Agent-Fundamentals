Module 1: Tool Calling

Exercise: Build a Financial Data Toolkit

Build a complete toolkit of 5 tools for a financial data system. This is the capstone for Module 1 — the toolkit you build here will be used in Modules 2 and 3.

Requirements

  1. Define 5 tools with precise descriptions and robust schemas:
    • get_firm_profile — basic firm information by CRD number
    • search_violations — search violation history with filters
    • check_compliance_status — current compliance status by rule category
    • get_regulatory_filing — retrieve a specific filing by type and firm
    • calculate_risk_score — compute a risk score based on firm data
  2. Wire up the full tool-calling loop — handle the complete request/response cycle.
  3. Handle at least 2 error scenarios — firm not found, invalid filing type, etc.
  4. Test with 3 queries that exercise different tools.

Validation criteria

  • All 5 tools defined with descriptions and schemas (required/optional params)
  • Full tool-calling loop works for 3 test queries
  • Error handling works for 2 failure scenarios
Build it: Use the sandbox below. The starter code provides the mock data and tool execution framework. You implement the tool definitions, the calling loop, and error handling.
This toolkit is your foundation for the rest of the course. In Module 2, you'll decide which problems need agents vs. functions. In Module 3, you'll plug these tools into a ReAct agent.
