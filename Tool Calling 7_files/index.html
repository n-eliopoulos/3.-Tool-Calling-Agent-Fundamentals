Module 1: Tool Calling

Multiple Tools

Real systems give Claude access to multiple tools. Claude reads all tool descriptions and selects the most appropriate one, or none, if it can answer directly.

How Claude selects tools

  • Description matching: Claude compares the user's query against each tool's description to find the best fit.
  • No tool needed: If Claude can answer from context alone, it responds with text — no tool call.
  • Overlapping tools: If two tools could work, Claude picks the one whose description best matches. This is why boundary conditions in descriptions matter.
tools = [
    {"name": "get_firm_profile", "description": "Look up basic firm info by CRD number..."},
    {"name": "get_recent_violations", "description": "Get violation history for a firm..."},
    {"name": "check_registration_status", "description": "Check if a firm or rep is currently registered..."},
]

# "Tell me about firm 12345" → get_firm_profile
# "Any violations for firm 12345?" → get_recent_violations
# "Is rep #67890 still registered?" → check_registration_status
# "What is FINRA?" → no tool call (general knowledge)
Try it: The sandbox defines 3 FINRA tools. Send queries that require different tools. Then send a query that needs no tools and observe Claude responding directly.

Keeping tool sets focused

More tools means more for Claude to evaluate. In practice, keep tool sets under 10–15 tools. If you need more, use a routing layer to select a relevant subset before sending to Claude.

Claude's tool selection is driven by descriptions. With well-described tools, it reliably picks the right one — even from a set of similar-sounding options.
