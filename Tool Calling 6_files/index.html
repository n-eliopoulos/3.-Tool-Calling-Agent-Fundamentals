Module 1: Tool Calling

Building Your First Tool End-to-End

Time to wire up the complete loop: define a tool → send a request → parse the tool_use block → execute a Python function → return the tool_result → get Claude's final response.

The complete pattern

# 1. Define the tool
tools = [{"name": "lookup_regulatory_filing", ...}]

# 2. Send request with tools
response = client.messages_create(tools=tools, messages=[...])

# 3. Check for tool_use in response
for block in response.content:
    if block.type == "tool_use":
        # 4. Execute the actual function
        result = execute_tool(block.name, block.input)
        # 5. Return tool_result
        messages.append({"role": "user", "content": [
            {"type": "tool_result", "tool_use_id": block.id, "content": result}
        ]})

# 6. Get final response
final = client.messages_create(tools=tools, messages=messages)
Guided exercise: In the sandbox, build a lookup_regulatory_filing tool that takes a filing_type and firm_id, returns mock filing data, and wire up the full loop. You must complete each step — this is a gated exercise.

What to watch for

  • The tool_use_id in the result must match the id from the tool_use block.
  • The tool result content must be a string (JSON-serialize objects).
  • Always check response.stop_reason — if it's "tool_use", Claude expects you to execute and return results.
This six-step pattern is the foundation of every tool-calling system. Once you can wire up one tool end-to-end, adding more tools is just repetition.
