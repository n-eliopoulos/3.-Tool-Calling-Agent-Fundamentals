ReAct: Reason + Act

ReAct is the foundational agent pattern. Each iteration has three phases: Thought (Claude reasons about what to do next), Action (Claude selects a tool call), Observation (tool result is returned). Repeat until done. The pattern degrades when an early Thought is wrong: an inaccurate intermediate conclusion shapes the next Action, which shapes the next Observation interpretation, and the error compounds across iterations rather than self-correcting.

The ReAct cycle

# System prompt instructs Claude to think before acting
system = """You are a regulatory investigation agent.
Before each action, emit your reasoning as a Thought.
Use the available tools to gather data.
When you have enough information, provide your final answer."""

# The loop
messages = [{"role": "user", "content": "Investigate firm 12345"}]

while True:
    response = client.messages_create(
        system=system, tools=tools, messages=messages
    )

    if response.stop_reason != "tool_use":
        break  # Final answer — no more tool calls

    # Execute tool calls, append results
    messages.append({"role": "assistant", "content": response.content})
    results = execute_tools(response)
    messages.append({"role": "user", "content": results})

Why "Thought" matters

The system prompt instructs Claude to reason before acting. This isn't just for readability — it improves tool selection accuracy. When Claude explains why it's calling a tool, it makes better choices.

Try it: Implement a minimal ReAct loop in the sandbox. The system prompt instructs Claude to emit reasoning before each tool call. Watch the Thought → Action → Observation cycle unfold.
ReAct = Reason + Act. The "Reason" step isn't decoration — it's a prompt engineering technique that improves the quality of the "Act" step.
