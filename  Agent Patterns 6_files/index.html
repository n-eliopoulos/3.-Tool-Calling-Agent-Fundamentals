State Management

The message history is the state. But it grows with every iteration. Managing context window budget — what to keep, what to summarize, what to drop — is critical for agents that run more than a few iterations.

Implicit vs. explicit state

  • Implicit (message history): Everything Claude has seen is in the messages array. Simple, but grows linearly. By iteration 5, you're sending thousands of tokens of history.
  • Explicit (state object): A separate dict that tracks what the agent has done and learned. Compact, queryable, and doesn't grow with message history.
state = {
    "tools_called": [],
    "results_received": {},
    "remaining_budget": 10000,
    "findings": [],
}

# Use state to make decisions
if "get_firm_profile" in state["tools_called"]:
    # Already have the profile — skip redundant calls
    pass

Context window management

  • Summarize old results: After N iterations, replace detailed tool results with summaries.
  • Drop redundant messages: If a tool was called and the result was "not found," you don't need the full exchange in history.
  • Keep the system prompt and goal: These anchor Claude's behavior across iterations.
Try it: Add a state dict to the ReAct agent that tracks tools called, results received, and remaining budget. Use it to avoid redundant tool calls.
Explicit state objects give you control over what the agent "remembers" without relying on the growing message history. Use both: messages for Claude, state for your code.
