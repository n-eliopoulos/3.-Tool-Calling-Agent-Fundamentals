Memory: Short-Term and Long-Term

Short-term memory is the current conversation. Long-term memory persists across sessions. Both are essential for agents that handle complex, multi-session tasks.

Short-term memory

The message history. It grows with every interaction. When it gets too long, summarize:

def summarize_history(messages, max_messages=20):
    if len(messages) <= max_messages:
        return messages
    # Keep system prompt + last N messages
    # Summarize the rest into a single message
    old = messages[1:-max_messages]
    summary = client.messages_create(
        messages=[{"role": "user", "content": f"Summarize: {old}"}]
    )
    return [messages[0], {"role": "user", "content": summary}, *messages[-max_messages:]]

Long-term memory

Key facts extracted after each task, persisted between runs:

memory = {
    "firm_12345": {
        "last_checked": "2024-10-15",
        "known_violations": 3,
        "risk_level": "medium",
    }
}

# Before calling a tool, check memory
if firm_id in memory and memory[firm_id]["last_checked"] == today:
    # Use cached data instead of calling the tool again
    pass

Short-term memory is automatic (message history) but needs management. Long-term memory is manual but enables agents that learn across sessions. Level 200 covers advanced memory patterns.
