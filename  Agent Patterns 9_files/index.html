Error Handling in Agent Loops

Agent loops have failure modes that single-pass functions don't: tool failures mid-loop, malformed responses, context window exhaustion, and infinite loops. Each needs specific mitigation.

Failure modes and mitigations

FailureMitigation
Tool throws exception mid-loopCatch, return error as tool_result, let Claude adapt
Claude returns unparseable responseRetry once, then abort with partial results
Context window exhaustionSummarize history before it hits the limit
Infinite loop (Claude keeps calling same tool)Track call history, break if same tool called 3+ times

Graceful degradation

def run_agent_safe(goal, tools, tool_fns, max_iterations=5):
    messages = [{"role": "user", "content": goal}]
    partial_results = []

    for i in range(max_iterations):
        try:
            response = client.messages_create(tools=tools, messages=messages)
        except Exception as e:
            return {"status": "error", "partial": partial_results, "error": str(e)}

        if response.stop_reason != "tool_use":
            return {"status": "complete", "answer": extract_text(response)}

        for block in response.content:
            if block.type == "tool_use":
                try:
                    result = tool_fns[block.name](**block.input)
                    partial_results.append({"tool": block.name, "result": result})
                except Exception as e:
                    result = f"Error: {e}"
                # Always return result (success or error)

    return {"status": "max_iterations", "partial": partial_results}
