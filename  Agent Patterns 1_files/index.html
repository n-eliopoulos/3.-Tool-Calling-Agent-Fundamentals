Module 3: Agent Patterns (Framework-Free)

Why Framework-Free First?

Frameworks hide the mechanics. When something breaks — and it will — you need to understand the loop underneath. We build from scratch first. Then frameworks in Level 200 make sense because you know what they're abstracting.

What frameworks abstract away

  • The agent loop: The while loop that drives think → act → observe → repeat.
  • Message history management: Appending assistant and tool_result messages in the right order.
  • Tool dispatch: Routing tool_use blocks to the right Python function.
  • Stop conditions: Deciding when the agent is done.
  • Error recovery: Handling tool failures mid-loop.

The problem with starting with frameworks

If your first agent is built with LangGraph or Strands, you'll know how to configure the framework — but not how to debug it when it behaves unexpectedly. Every framework bug eventually requires understanding the underlying loop.

Program principle: This course builds everything from scratch using Claude's native tool_use API. No LangChain, no LangGraph, no Strands. Those come in Track 3 Level 200, after you understand what they're doing under the hood.
Framework-free first isn't anti-framework. It's pro-understanding. Build it once from scratch, then use frameworks with confidence.
