The Agent Loop

Every agent — regardless of framework — implements the same core pattern: while not done: think → act → observe → update state. The differences between frameworks are in how they manage state, select actions, and decide when to stop.

The loop in pseudocode

messages = [{"role": "user", "content": goal}]

while not done:
    # THINK: Claude reasons about what to do next
    response = claude(messages, tools)

    # CHECK: Is Claude done? (no more tool calls)
    if response.stop_reason != "tool_use":
        done = True
        break

    # ACT: Execute the tool(s) Claude requested
    results = execute_tools(response)

    # OBSERVE: Add results to message history
    messages.append(assistant_message(response))
    messages.append(tool_results_message(results))

    # UPDATE: Increment iteration counter, check budget
    iterations += 1

What varies between implementations

  • State: Just message history? Or an explicit state object?
  • Stop conditions: Claude decides? Max iterations? Token budget?
  • Error handling: Retry? Skip? Abort?
  • Observability: What gets logged at each iteration?
The agent loop is simple: think → act → observe → repeat. Everything else — frameworks, patterns, architectures — is variation on this core loop.
