Tool Descriptions: The Prompt Engineering of Tools

The description field is a prompt. Claude reads it to decide when to call a tool, when not to call it, and what to expect back. Vague descriptions lead to wrong tool selection. Precise descriptions lead to reliable behavior.

Vague vs. precise

# ❌ Vague — Claude doesn't know when to use this
"description": "Gets information about a firm"

# ✅ Precise — Claude knows exactly when and how
"description": "Look up a FINRA-registered firm's profile by CRD number. Returns firm name, registration status, address, and number of registered representatives. Use this when the user asks about a specific firm's basic information. Do NOT use for violation history or compliance status — use get_recent_violations or check_compliance_status instead."

What makes a good tool description

  • What it does — specific action and data returned.
  • When to use it — the types of queries that should trigger this tool.
  • When NOT to use it — boundary conditions that prevent wrong selection.
  • What it returns — the shape and content of the response.
