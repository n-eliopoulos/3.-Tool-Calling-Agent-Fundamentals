Module 2: Agent vs. Function

The Cost of Agent Complexity

Every layer of autonomy has a price. More LLM calls means more cost and latency. Loops can run away. State management adds code. Debugging is harder. Testing is harder. Quantify these costs before choosing an agent.

The math

ApproachLLM CallsLatencyCost per Task
Smart function (no tools)11–3s$0.003–0.01
Smart function (with tools)23–6s$0.01–0.03
Bounded agent (3 iterations)4–78–20s$0.03–0.10
Autonomous agent (unbounded)5–20+15–60s+$0.10–1.00+

Hidden costs

  • Context window growth: Each iteration adds messages. By iteration 5, you're sending the full history — input tokens compound.
  • Debugging time: A function fails the same way every time. An agent fails differently each run. Non-deterministic debugging is expensive.
  • Testing complexity: Testing a function: mock inputs, assert outputs. Testing an agent: mock tool sequences, handle branching paths, verify stop conditions.
  • Runaway risk: A bug in a function wastes one API call. A bug in an agent loop can burn through your token budget.
