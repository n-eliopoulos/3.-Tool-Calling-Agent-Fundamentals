Planning-Execution Pattern

A two-phase agent: first Claude generates a plan, then it executes each step. If a step fails, Claude re-plans from the current state. More structured than pure ReAct.

Phase 1: Planning

plan_prompt = f"""Given this goal: {goal}
And these available tools: {tool_descriptions}
Generate a step-by-step plan as JSON:
[{{"step": 1, "tool": "tool_name", "reason": "why this step"}}]"""

plan = client.messages_create(messages=[{"role": "user", "content": plan_prompt}])
steps = json.loads(plan.content[0].text)

Phase 2: Execution

for step in steps:
    try:
        result = execute_tool(step["tool"], step.get("args", {}))
        state["completed"].append({**step, "result": result})
    except Exception as e:
        # Re-plan from current state
        new_plan = replan(goal, state["completed"], str(e), tools)
        steps = new_plan  # Continue with updated plan
        break

When to use plan-then-execute

  • Predictable tasks with known steps: The plan provides structure and auditability.
  • Tasks requiring user approval: Show the plan before executing — the user can modify it.
  • Debugging: A plan is easier to inspect than a ReAct trace.
Try it: Implement plan-then-execute in the sandbox. Phase 1: Claude generates a JSON plan. Phase 2: Execute each step. If a step fails, ask Claude to re-plan from the current state.
Plan-then-execute trades flexibility for structure. Use it when you want auditability, user approval, or predictable execution order.
