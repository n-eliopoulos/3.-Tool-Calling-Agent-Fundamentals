Tool Calling & Agent Fundamentals

Module 1: Tool Calling

What Is Tool Calling?

LLMs can't execute code, query databases, or access external systems. They generate text. That's it. Tool calling bridges this gap: the model decides what to call and with what arguments, and your code executes it. The model never touches your systems directly.

Key insight: The LLM is the decision-maker. Your code is the executor. This separation is fundamental — the model reasons about what action to take, but all actual execution happens in code you control.

The mental model

  • Without tools: User asks a question → LLM generates an answer from training data (may be wrong or outdated).
  • With tools: User asks a question → LLM decides it needs data → requests a tool call → your code executes it → result goes back to the LLM → LLM generates an informed answer.

Why this matters at FINRA

Regulatory data changes constantly — firm registrations, violation histories, filing statuses. An LLM can't know the current compliance status of firm CRD#12345. But it can decide to call check_compliance_status(firm_id="12345") and use the live result to answer the question.

Tool calling turns an LLM from a text generator into a reasoning engine that can interact with real systems — while keeping your code in full control of what actually executes.
