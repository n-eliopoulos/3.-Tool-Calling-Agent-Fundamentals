Stop Conditions and Runaway Prevention

An agent without stop conditions burns money. Every production agent needs at least two stop strategies.

Three stop strategies

  1. Claude decides it's done: No more tool_use blocks — Claude responds with text only. This is the natural stop condition.
  2. Max iteration limit: Hard cap on loop iterations. If Claude hasn't finished in N iterations, stop and return partial results.
  3. Token budget tracking: Track cumulative input + output tokens. Stop before exceeding a cost threshold.

Always implement at least two

for i in range(max_iterations):
    response = client.messages_create(...)
    total_tokens += response.usage.input_tokens + response.usage.output_tokens

    if response.stop_reason != "tool_use":
        break  # Strategy 1: Claude is done

    if total_tokens > token_budget:
        break  # Strategy 3: Budget exceeded

# Strategy 2: max_iterations enforced by the for loop
Try it: Add max iteration guard and token budget tracking to the ReAct agent. Set max_iterations=2 on a task that needs 4 steps. Observe graceful degradation — the agent returns what it found so far.
Never deploy an agent without a max iteration limit. It's the seatbelt of agent development — you hope you never need it, but when you do, it saves you.
