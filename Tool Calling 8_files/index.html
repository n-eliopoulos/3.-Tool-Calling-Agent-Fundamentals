Parallel Tool Calling

Claude can request multiple tool calls in a single response when it needs independent pieces of data. Instead of one tool_use block, you get two or more — each with its own id, name, and input.

When Claude calls tools in parallel

  • The user asks about multiple independent entities: "Compare compliance status of firms A and B."
  • The query needs data from multiple sources that don't depend on each other.
  • Claude recognizes the data needs are independent and batches the calls.

Handling parallel calls

# Claude returns multiple tool_use blocks
tool_calls = [b for b in response.content if b.type == "tool_use"]

# Execute all of them (can be done in parallel)
results = []
for call in tool_calls:
    result = execute_tool(call.name, call.input)
    results.append({
        "type": "tool_result",
        "tool_use_id": call.id,
        "content": result
    })

# Return ALL results in a single message
messages.append({"role": "user", "content": results})
Try it: Send "Compare the compliance status of firms A and B" in the sandbox. Claude returns two tool_use blocks. Execute both and return the results together.
Always check for multiple tool_use blocks in a response. Return all results in a single message — Claude needs all of them to generate its final answer.
