Smart Functions: The 80% Solution

A smart function calls an LLM with tools, handles errors, retries on failure, and returns structured output. No loop, no planning, no memory. Single pass. Handles most real-world tasks.

The pattern

def analyze_filing(filing_id):
    """Smart function: single-pass tool calling with error handling."""
    filing = get_filing(filing_id)

    response = client.messages_create(
        model="claude-sonnet-4",
        system="Analyze this regulatory filing. Return structured JSON.",
        tools=[rule_lookup_tool, firm_profile_tool],
        messages=[{"role": "user", "content": f"Analyze filing: {filing}"}],
    )

    # Handle tool calls (single round)
    if response.stop_reason == "tool_use":
        results = execute_all_tools(response)
        response = client.messages_create(
            tools=[rule_lookup_tool, firm_profile_tool],
            messages=[..., results],
        )

    return parse_json(response.content[0].text)

Why this is usually enough

  • Predictable cost: At most 2 LLM calls (initial + after tool results).
  • Predictable latency: Bounded by the number of tool calls, which you control.
  • Easy to test: Input → output. Mock the tools, assert the output.
  • Easy to debug: Linear execution path. No loops to trace.
Try it: Build an analyze_filing smart function in the sandbox. Single pass — no loop. Use 2 tools, handle errors, return structured output.
Before reaching for an agent, ask: "Can a single-pass function with tools solve this?" If yes, stop there. You'll save cost, latency, and debugging time.
