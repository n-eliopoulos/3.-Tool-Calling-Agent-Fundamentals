Exercise: Full ReAct Agent for Regulatory Investigation

Build a complete ReAct agent that investigates a firm: gathers profile data, checks violations, reviews filings, assesses compliance risk, and produces a structured investigation report. This is the capstone for the entire course.

Requirements

  1. Uses all 5 tools from the Module 1 financial toolkit.
  2. ReAct loop with Thought → Action → Observation cycle.
  3. Max iteration guard — stops gracefully if limit is reached.
  4. State tracking — explicit state dict tracking tools called and findings.
  5. Structured logging — captures every iteration's thought, action, and result.
  6. Error handling — graceful degradation when a tool fails mid-run.
  7. Structured output — final report as JSON with findings and risk assessment.

Validation criteria

  • Agent completes investigation using 3+ tools
  • Max iteration guard works (test with low limit)
  • State is tracked across iterations
  • Logging captures all iterations
  • Error handling works (shim simulates 1 tool failure mid-run)
  • Final report is structured JSON with findings and risk assessment
Build it: This exercise ties together everything from all 3 modules: tool definitions (Module 1), the decision to use an agent (Module 2), and the ReAct pattern with all production safeguards (Module 3).
A production-ready agent is the ReAct loop plus guardrails: stop conditions, state management, error handling, and observability. The loop is 30 lines. The guardrails are the other 70.
