Module 2: Agent vs. Function

The Spectrum of Autonomy

Agent vs. function isn't a binary. It's a spectrum from fully deterministic code to fully autonomous agents. Each step adds capability and complexity. Most production systems live in the middle.

Five levels

  1. Pure function: Deterministic code. Input → output. No LLM involved. Example: validate a CRD number format.
  2. LLM function: Single LLM call, no tools. Input → prompt → response. Example: classify a customer complaint.
  3. Tool-calling function: Single LLM call with tools. One round of tool use, then done. Example: look up a firm and summarize its profile.
  4. Bounded agent: Loop with tools, but bounded by max iterations or a fixed plan. Example: investigate a firm across 3 data sources.
  5. Autonomous agent: Open-ended loop. Decides its own plan, adapts strategy, runs until it decides it's done. Example: conduct a full regulatory examination.
Try it: The interactive spectrum below lets you click each level to see its definition, a FINRA example, complexity cost, and when to use it.

The key insight

Level 3 (tool-calling function) handles the vast majority of real-world tasks. Levels 4–5 are powerful but expensive, slow, and hard to debug. Don't reach for an agent when a function will do.

Start at the lowest level of autonomy that solves the problem. Move up only when you hit a limitation that requires more flexibility.
